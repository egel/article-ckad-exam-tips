\subsection{CKAD vs CKA: which exam is for me?}

In this section, I will cover preparation for the CKAD exam. Initially, my first questions were about which course or exam I should aim for: CKA, CKAD, or something else? Let's clarify what CKAD and CKA are.

\begin{definition}[CKAD] 
	Certified Kubernetes Application Developer is a certification that focuses on skills required for successful Kubernetes application development. It assumes prior knowledge of container runtimes and microservices, and validates the ability to work with Kubernetes resources and Cloud Native concepts.~\cite{linuxfoundation-ckad}
\end{definition}

CKAD is primarily designed for developers who are actively working and whose sole purpose is to build and containerize software to run within a Kubernetes environment. This course does not sink deep into cluster administration. Instead, it covers the general theory and involves shallow administrative tasks such as enabling plugins in the cluster (e.g., \href{https://kubernetes.io/docs/reference/access-authn-authz/rbac/}{Role-Based Access Control (RBAC)}).

\begin{definition}[CKA]
	The Certified Kubernetes Administrator certification focuses on skills for administering and managing Kubernetes clusters. It assumes prior knowledge of container runtimes and basic Kubernetes concepts. The certification validates the ability to deploy, manage, and maintain Kubernetes clusters.~\cite{linuxfoundation-cka}
\end{definition}

\subsection{Courses and Exam Preparation}
Before we begin, let's address some common questions:
\begin{itemize}
	\item Do I need a course? Which one should I choose?
	\item Where can I purchase the exam?
	\item How do I start learning Kubernetes?
\end{itemize} 

\paragraph{Do I need a course?} No, there is no mandatory procedure to buy a course to learn Kubernetes or pass an exam. You can simply learn on your own with the resources you find online. For example, the foundation of almost everything you will learn can be found at CKAD curriculum~\cite{cncf-curriculum} and offical Kubernetes documentation at \href{https://kubernetes.io/docs}{https://kubernetes.io/docs}.

Although buying a course can help you prepare, as it usually provides some structure while learning and gradually introduces new concepts from basic to advanced.

\begin{note}{note:cncf-curriculum}
	When planning to learn on your own and later purchase an exam, remember to follow the Curriculum for CKAD~\cite{cncf-curriculum}. Next, while learning, give yourself a structure and review all the topics you've covered. With this you will be sure you did not miss a thing or learn a topic that is meant for another course (e.g, CNA).
\end{note}


\paragraph{Which course should I choose?} There are plenty of resources available online for free and from my experience, there is no need to buy a course. However, if you have already decided that you want to purchase one, there are many options available.

At the time when I was looking to buy a course for myself, I received a recommendation from a friend and did not particularly search for the best or most recommended one. I had some budget to spend, so I simply purchased a bundle \href{https://trainingportal.linuxfoundation.org/collections/kubernetes-developer}{(course LFD259 + Exam)} directly from the Linux Foundation. I bought it at that time because there was 40\% off the base price.\footnote{Later, I found out that the Linux foundation often has sales and discounts of 20-40\%. You can subscribe to their newsletter to receive information about deals via email.}


\paragraph{LFD259 from Linux Foundation} - I took this course and found it to be average.\footnote{Although after researching alternative options, I found it to be overpriced for what you get. On the other hand, being an official course ensures that the content is always up-to-date with the current Kubernetes Curriculum.}

The course covers all the basics, but some topics may require more in-depth study since the content is mostly text-based and doesn't provide all the nitty-gritty details. There are only a few small videos that explain more detailed concepts.

The course material is divided into chapters with dry theory, followed by exercise labs and homework tasks at the end of each chapter. I found the labs to be the most engaging part of the course as they involve real-world exercises to build something with your cluster.\footnote{For example, I enjoyed the labs where you create a basic registry in your cluster to upload Docker images later on. However, it's worth noting that at the CKAD exam, you won't need to build the registry from scratch - at most, you'll need to push an image to a given registry URL that's already set up.}

While I found the labs to be a great way to get hands-on experience with Kubernetes and develop good habits, they didn't provide much practice material directly applicable to the CKAD exam (though I'll cover this in more detail in \nameref{sec:focus-on-exam}).

\paragraph{Where to buy the exam?} You can purchase an official CKAD exam directly from the Linux Foundation shop \burl{https://training.linuxfoundation.org/certification/certified-kubernetes-application-developer-ckad/}. Later, I found a webpage \url{https://kube.promo} that offers various Kubernetes Promo codes with great discounts where you might find something that suite you better.

\begin{tip}{tip:buyexam1}
	If you want to buy a bundle \texttt{course} + \texttt{exam}, be prepared to have only 12 months from the date of purchase, during which you must schedule your final exam date\footnote{For me 12 month initially sounded like a lot of time, but the course and exam is quite challenging, and if you have regular work, family, kids, the time goes by really fast}. Plan your learning sessions across weeks to enjoy the process and avoid rushing when the course deadline approaches.\footnote{I personally experienced this situation, where I bought the course but didn't learn much for a few initial months, then had to speed up intensely due to reminders 3 months prior to the course expiration, which forced me to change my schedule and refocus on the course.}
\end{tip}

\begin{tip}{tip:not-learn-by-heart}
	It is also vital to note that you do not need to know a complete definition of each term or component. Instead, focus on understanding what they are and where they are used in context. In the CKAD exam, you have only practical tasks where you will need to apply this knowledge dirtectly on the cluster, and I am not aware of any questions requiring writing a definition -- only practical examples.
\end{tip}

\subsection{Setup infrastucture}
\label{subsec:setup-infrastructure}

There are few ways how you can start with practical learning of Kubernetes. I found there are 3 the most popular options you can choose from:


\paragraph{Rent the Cluster} You can simply buy some online cluster and use it for the learning purposes. Most popular options would be (order does not matter): 
\begin{enumerate}
	\item Google Cloud Kubernetes Engine (GKE)
	\item Amazon Elastic Container Service for Kubernetes (EKS)
	\item Microsoft Azure Kubernetes Service (AKS)
\end{enumerate}


\begin{tabularx}{\textwidth}{|X|X|}
	\hline
	\textbf{Advantages} & \textbf{Disadvantages} \\
	\hline
	\begin{enumerate}
		\item minimal time required to set up a Kubernetes environment
		\item when needed start learning almost instantly
		\item no need to deep configuration as managed by cloud providers
	\end{enumerate}
	
	&
	
	\begin{enumerate}
		\item additional costs as this type of services are usually paid
		\item commonly you pay for usage / time, so also need to remember shutdown service in order your bill will not explode
		\item no need to deep configuration as managed by cloud providers
	\end{enumerate}\\
	\hline
\end{tabularx}


\paragraph{Minikube} For some basic learnings you can just use your computer and setup localy a minikube. This is great for inital explore or for some simple testing, but there are some limitations of this appoach for example if want to simulate multiple nodes.\\

\begin{tabularx}{\textwidth}{|X|X|}
	\hline
	\textbf{Advantages} & \textbf{Disadvantages} \\
	\hline
	\begin{enumerate}
		\item no additional costs
		\item work on single computer
		\item easy to work \& learn
		\item after downloading and setup can work offline\footnote{Of course if you may need to download some additional images you will require internet, but in general after this basic setup you should be able to learn offline}
		\item can start learning almost instantly
		\item little to none configuration as this is prepared package
	\end{enumerate}
	
	&
	
	\begin{enumerate}
		\item only run with single node (control plane)
		\item cannot simulate all scenarios especially when learning new about multiple nodes
		\item allowed but not recommented for exam environment
	\end{enumerate}\\
	\hline
\end{tabularx}

\paragraph{Bild own cluster} If you dare to explore kubernetes in full depth you can buy some hardware and setup your own cluster at home. I took this approach as wanted and plan to explore kubernetes the hard-way. Although, honestly, I would not recommend this approach if you just want to learn to the exam and pass.\\

\begin{tabularx}{\textwidth}{|X|X|}
	\hline
	\textbf{Advantages} & \textbf{Disadvantages} \\
	\hline
	\begin{enumerate}
		\item complete control of the infrastrucute
		\item great to work / learn kubernetes
		\item little to none configuration as this is prepared package
		\item demanding but very satisfying 
	\end{enumerate}
	
	&
	
	\begin{enumerate}
		\item additional cost of hardware
		\item setup may take from few hours (for expert) to few days (begginer) depends on you current skills
		\item need learn much more then just to pass CKAD exam, as you will assembe and setup own cluster from scratch
		\item not recommened if you just want pass CKAD
		\item not recommened if you do not want to own and maintian cluster
	\end{enumerate}\\
	\hline
\end{tabularx}
